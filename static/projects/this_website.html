This Website

This is a website for purists.

I am frequently frustrated by the ineficiency and fragmentation of modern web development. To implement client side routing for a more responsive user experience, for example, you would typically be encouraged to adopt a JavaScript framework like Vue. Let me walk you though my experience with that:

  1. Opened up the Vue v2 Guide
  2. Realized that Vue has its own ways to do:
    • Component registration (and communication)
    • Event handling
    • Forms
    • Unit testing
    • Etc.
  3. Looked for slimmer alternatives (oh hey, maybe react?).

You can imagine how my React experience went.

My beef with this situation wasn't an aversion to learning -- it was a fear that after adopting one of these technoligies, I might be stuck with it. What if it's no longer maintained in 5 years? What if I need help maintaining it, but can't find people who know my specific "stack" of frameworks anymore? What if I just want a different feature set? Commitment is scary, and web development is full of it.
