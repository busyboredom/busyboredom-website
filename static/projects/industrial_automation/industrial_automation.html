Industrial Automation

Summary

I went into industrial automation out of college, so this is a bit more than one project. It's how I paid my loans off, and oh boy does it take a lot of programming to pay off a degree nowadays. Luckily, industrial automation is a rewarding field to work in; it makes people's work go farther, so that someday we might not have to work so hard.

At Universal Machine and Engineering (UMEC), the role of a controls engineer is to program machines to do the tasks expected of them. Those tasks range from putting bottles in boxes to cutting up lobsters -- the client specifies the task, the mechanical guys build a machine, and the controls guys bring it to life.

This page lists a few of the industrial automation jobs I have had a hand in.

Autoinjector Sleever

This is the project shown in the video at the top of this page. The machine's task was simple:

  1. Print some info on a plastic sleeve.
  2. Put it on an injector.
  3. Shrink it on tight.
But as with all large projects, making it work isn't quite as easy as those 3 steps sound. To meet the client's neets, the machine also needed to:
  • Verify the information printed using optical character recognition.
  • Eject bad prints before shrinking.
  • Reject bad parts (for faulty shrinkage) after shrinking.
  • Load injectors from trays.
  • Allow continuous operation during loading and unloading.
  • Record temperature, speed, rejects, operator credentials, etc. in a database.
  • Do it all at 60 parts per minute.
There were many other requirements, but this was the bulk of it. We used two 6-axis Epson robots to load the injectors from trays inserted by an operator, Cognex cameras to read the print and inspect the shrinkage, pneumatic pistons for ejecting/rejecting parts and an Epson SCARA robot for unloading the sleeved injectors into bins. Data was pushed to Microsoft SQL Server while inspection images were sent to an FTP server, and it was all controlled by an Allen-Bradley PLC.

Vial Cartoner

This was my first project at UMEC. The task was to:

  1. Fold boxes (i.e. cartons).
  2. Verify the carton barcodes using an ID40 barcode scanner.
  3. Load vials into cartons two at a time.
  4. Fold the cartons shut.
  5. Reject bad parts (empty carton, bad barcode read, etc.).
  6. Place 60 vials per minute.
Pick-and-place for the vials was accomplished with an Epson SCARA robot, while rejecting was done with a blower at the end of the line. The unreliability of the carton infeed mechanism made speed the primary challenge, and meeting the specified rate required me to pre-fetch the vials and hover over position while waiting for the cartons to index. I got it up to over 80 parts per minute in the end, beating the requirement by a pretty wide margin.

Bottler Interlocks

This line does a lot. Bottles go though stations for:

  1. Sanitization.
  2. Filling.
  3. Capping.
  4. Inspection.
  5. Metal detection.
  6. Wrapping.
  7. Sealing.
  8. Labeling.
Most of the line was in place before I started work on it, but none of the components were talking to each other. The metal detector could go down and the capper would keep going like nothing happened, and that's a problem.

My role was to program a PLC to turn each component on and off according to certain criteria like "if the conveyor stops, raise the heat tunnel". While I was at it, I was also asked to add an HMI, program the post-capper inspection camera, and program the reject setup.
